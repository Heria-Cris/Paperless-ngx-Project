# Development Plan

## 阶段状态同步（2026-06-29）

- 阶段 0：已完成，项目骨架与运行入口。
- 阶段 1：已完成，基础可访问界面。
- 阶段 2：已完成，登录与页面访问控制。
- 阶段 3：已完成，数据库表与基础数据模型。
- 阶段 4：已完成，分类和标签真实 CRUD。
- 阶段 5：已完成，文档元数据 CRUD。
- 阶段 6：已完成，真实文件上传与下载。
- 下一阶段：阶段 7，建议完善搜索、筛选、分页与首页统计。

> 开发策略：先落地，再完善。第一阶段先做出一套基础可访问、可演示、可点击的界面骨架；第二阶段再逐步把页面接入真实业务、数据库、文件存储和权限控制。

## 总体原则

- 先保证系统能运行、能访问、能演示整体流程。
- 前期页面可以使用静态数据或内存模拟数据，不急于一次性完成数据库和复杂业务。
- 优先完成主流程页面：登录、首页、文档列表、上传表单、分类管理、标签管理。
- 后续再逐步替换为真实数据库、真实文件上传下载、真实权限和日志。
- 每完成一个阶段，都保持系统可启动、可访问、可截图演示。

## 推荐开发阶段

### 阶段 0：项目骨架与运行入口

目标：创建一个可以启动的 Spring Boot Web 项目。

工作内容：

- 创建 Maven 项目结构。
- 引入 Spring Boot Web、Thymeleaf、Lombok、Validation 等基础依赖。
- 配置基础 `application.yml`。
- 创建首页 Controller。
- 创建基础模板目录。
- 启动后能访问一个简单首页。

验收标准：

- 执行 `mvn spring-boot:run` 或运行主类可以启动项目。
- 浏览器能访问首页。
- 项目目录结构清晰，后续成员可以开始并行开发。

### 阶段 1：先落地基础界面

目标：先做出一套可访问、可点击、视觉统一的后台管理界面。

这一阶段不强求真实业务逻辑，允许使用静态数据、模拟数据或临时 Controller 返回页面。

优先页面：

- 登录页。
- 后台首页 / 仪表盘。
- 文档列表页。
- 文档上传页。
- 文档详情页。
- 文档编辑页。
- 分类管理页。
- 标签管理页。
- 用户管理页。
- 操作日志页。

公共界面：

- 顶部导航。
- 侧边菜单。
- 页面标题区域。
- 表格样式。
- 表单样式。
- 按钮样式。
- 分页样式。
- 删除确认提示。
- 错误/成功提示区域。

验收标准：

- 所有核心页面 URL 都能访问。
- 菜单可以跳转到各模块页面。
- 页面风格统一，能作为课程演示的第一版界面。
- 文档列表、分类列表、标签列表可以展示模拟数据。
- 上传、编辑、搜索表单可以展示，但暂时可以不保存真实数据。

建议路由：

- `/login`
- `/logout`
- `/dashboard`
- `/documents`
- `/documents/upload`
- `/documents/{id}`
- `/documents/{id}/edit`
- `/categories`
- `/tags`
- `/users`
- `/logs`

### 阶段 2：接入基础登录与页面访问控制

目标：让系统从“页面可访问”进入“有登录状态”的基础系统。

工作内容：

- 创建用户实体和模拟用户数据，或先使用内存用户。
- 完成登录表单提交。
- 登录成功后写入 Session。
- 完成退出登录。
- 创建 HandlerInterceptor。
- 未登录访问内部页面时跳转到登录页。
- 页面根据当前用户显示用户名和角色。

验收标准：

- 未登录时不能访问后台页面。
- 登录后可以进入首页。
- 退出后 Session 清除。
- 管理员和普通用户角色在页面上可区分。

说明：

- 这一阶段可以暂时不接 MySQL。
- 密码哈希可以在数据库接入阶段完善。

### 阶段 3：数据库表与基础数据模型

目标：把系统核心数据结构确定下来，为真实功能做准备。

工作内容：

- 设计并创建数据库表。
- 配置 MySQL 数据源。
- 引入 MyBatis-Plus。
- 创建 Entity、Mapper、Service 基础结构。
- 准备初始化 SQL 或测试数据。

优先表：

- `sys_user`
- `document`
- `document_category`
- `document_tag`
- `document_tag_rel`
- `operation_log`

验收标准：

- 项目能连接 MySQL。
- MyBatis-Plus Mapper 可正常查询测试数据。
- 页面模拟数据可以开始替换为数据库数据。

### 阶段 4：分类和标签真实 CRUD

目标：先完成相对简单、独立的后台维护功能。

工作内容：

- 分类列表、新增、编辑、删除。
- 标签列表、新增、编辑、删除。
- 删除分类时检查是否被文档使用。
- 删除标签时解除文档标签关联。
- 分类和标签数据提供给文档上传/编辑表单使用。

验收标准：

- 管理员可以维护分类和标签。
- 文档表单能加载真实分类和标签。
- 普通用户不能进入分类和标签维护页面，或至少菜单不可见。

### 阶段 5：文档元数据 CRUD

目标：先完成文档“信息管理”，不急于处理真实文件。

工作内容：

- 文档列表从数据库读取。
- 文档详情展示数据库元数据。
- 文档新增或上传表单保存元数据。
- 文档编辑更新标题、分类、标签、描述。
- 文档删除先做逻辑删除。
- 普通用户只能查看和管理自己的文档。

验收标准：

- 文档列表、详情、编辑、删除形成闭环。
- 文档与分类、标签关联正常。
- 管理员能看全部文档，普通用户只能看自己的文档。

说明：

- 这一阶段可以先用占位文件路径。
- 真实文件上传下载放到下一阶段。

### 阶段 6：真实文件上传与下载

目标：把文档管理从“元数据”升级为“真实文件管理”。

工作内容：

- 实现 Multipart 文件上传。
- 校验文件类型。
- 校验文件大小。
- 使用唯一文件名保存到本地目录。
- 按日期和用户组织目录，例如 `uploads/2026/06/user_1/`。
- 数据库存储原始文件名、存储文件名、路径、大小、类型。
- 实现文件下载。
- 下载前校验权限。
- 文件不存在时给出友好提示。

验收标准：

- 用户可以上传真实文件。
- 上传后列表显示文件信息。
- 用户可以下载有权限访问的文件。
- 普通用户无法下载其他用户私有文档。

### 阶段 7：搜索、筛选、分页与首页统计

目标：提升系统可用性和演示完整度。

工作内容：

- 文档列表分页。
- 关键词搜索：标题、原始文件名、描述。
- 分类筛选。
- 标签筛选。
- 文件类型筛选。
- 上传时间筛选。
- 首页统计卡片。
- 最近上传文档列表。

验收标准：

- 文档列表支持组合查询。
- 列表分页正常。
- 首页展示管理员全局统计或普通用户个人统计。

### 阶段 8：基础操作日志

目标：补齐项目文档要求中的基础可追踪能力。

工作内容：

- 登录日志。
- 上传日志。
- 删除日志。
- 下载日志可选。
- 管理员日志列表。
- 按用户、操作类型、时间筛选可作为进阶完善。

验收标准：

- 关键操作能写入日志表。
- 管理员可以查看日志列表。

### 阶段 9：进阶功能与答辩优化

目标：在 MVP 稳定后选择性增强展示效果。

可选功能：

- 回收站恢复。
- PDF 预览。
- 图片预览。
- TXT 预览。
- 批量删除。
- 批量修改分类。
- 批量添加标签。
- 文件重复检测。
- 存储空间统计。
- 页面样式细节优化。

验收标准：

- MVP 主流程稳定。
- 至少 1 到 3 个进阶功能可演示。
- 页面和答辩演示流程顺畅。

## 详细开发顺序

1. 创建 Spring Boot 项目骨架。
2. 做登录页、后台布局、侧边菜单和首页。
3. 做文档列表、上传、详情、编辑页面，先使用模拟数据。
4. 做分类、标签、用户、日志页面，先使用模拟数据。
5. 接入 Session 登录和登录拦截。
6. 接入 MySQL 和 MyBatis-Plus。
7. 完成分类、标签真实 CRUD。
8. 完成文档元数据 CRUD。
9. 完成文档与标签的多对多绑定。
10. 完成真实文件上传。
11. 完成真实文件下载。
12. 完成文档权限校验。
13. 完成搜索、筛选和分页。
14. 完成首页统计。
15. 完成基础操作日志。
16. 选择性完成回收站、预览、批量操作等进阶功能。
17. 统一页面样式，整理演示数据和答辩流程。

- A：权限拦截和后台菜单权限。

## 每阶段完成定义

每个阶段完成时至少满足：

- 项目可以启动。
- 已完成页面可以访问。
- 不影响已有页面。
- 新增功能有基本异常处理。
- `progress.md` 已更新。
- 如涉及数据库，初始化 SQL 或表结构说明已同步。
