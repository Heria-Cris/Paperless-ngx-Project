# Project Context

> 本文件是后续对话和开发任务的固定项目背景。每次继续开发时，优先阅读本文件，再结合 `progress.md` 判断当前进度与下一步工作。

## 项目定位

本项目是一个课程实训用的“本地文档管理系统”，参考 paperless-ngx 的核心思想，但进行大幅裁剪，目标是实现一个适合 Java Web 课程展示和 5 人团队协作开发的简化版文档管理平台。

系统用于管理本地电子文档，例如课程资料、合同、证书扫描件、发票、报告等，解决传统文件夹管理中分类混乱、检索困难、权限缺失、文件重复、操作不可追踪等问题。

## 核心目标

- 实现典型 Java Web CRUD 项目开发闭环。
- 支持文档上传、存储、下载、预览和基础管理。
- 支持文档分类、标签、多条件查询和分页。
- 支持登录、Session 会话、管理员与普通用户的基础权限控制。
- 支持首页统计和基础操作日志。
- 保持单体架构清晰，便于 5 人并行开发和课程答辩演示。

## 技术栈约束

### 后端

- Java 17
- Spring Boot 3
- Spring MVC
- Thymeleaf
- Session 登录状态管理
- HandlerInterceptor 登录拦截与简单权限控制
- MyBatis-Plus
- Spring Validation
- Lombok
- Maven
- Hutool

### 前端

- 服务端渲染为主，不做完整前后端分离。
- HTML5、CSS3、JavaScript。
- Thymeleaf 渲染页面数据。
- Bootstrap 或 Element Plus 风格可作为后台管理界面样式参考。
- 可选使用简单 Vue、Axios 或 Fetch 做局部交互，例如标签选择、上传预览、删除确认、统计刷新。

### 数据库与存储

- MySQL 8 存储用户、角色、文档元数据、分类、标签、日志等结构化数据。
- 本地文件系统保存真实上传文件。
- 数据库只保存文件路径、原始文件名、元数据和关联关系。
- 基础搜索使用 MySQL LIKE，后期可选 MySQL FULLTEXT。

## 明确不做

基础版本不实现以下能力：

- OCR 文字识别。
- Redis、异步任务队列、任务调度。
- Elasticsearch。
- 邮件自动导入。
- 机器学习自动分类。
- 复杂工作流。
- 多租户系统。
- 对象级复杂权限或完整 RBAC。
- Vue Router、Pinia、Token 鉴权等完整前后端分离体系。

## 架构方式

采用 Spring Boot 单体 Web 架构。

建议分层：

- `Controller`：页面跳转、表单提交、上传下载请求。
- `Service`：业务逻辑、权限判断、文件和数据库协同处理。
- `Mapper` / `DAO`：数据库访问。
- `Entity`：数据库实体。
- `DTO` / `VO`：表单参数和页面展示数据。
- `View`：Thymeleaf 模板页面。
- `Interceptor`：登录拦截和基础角色权限控制。
- `Exception` / `Result`：统一异常和返回信息。

## 用户角色

### 管理员

管理员拥有系统最高权限：

- 管理所有用户。
- 查看、上传、修改、删除所有文档。
- 管理文档类型和标签。
- 查看操作日志。
- 查看系统统计信息。
- 恢复或彻底删除回收站文档。

### 普通用户

普通用户主要管理个人文档：

- 登录系统。
- 上传个人文档。
- 查看、修改、删除自己上传的文档。
- 下载自己有权限访问的文档。
- 使用分类、标签、关键词筛选文档。
- 查看个人统计信息。

## 权限规则

- 未登录用户不能访问系统内部页面。
- 管理员可访问全部功能。
- 普通用户只能管理自己的文档。
- 普通用户不能访问用户管理、系统日志、类型维护等后台管理功能。
- 下载、预览、编辑、删除文档前必须校验当前用户权限。

### 文件上传、预览与下载

上传要求：

- 上传时填写文档标题。
- 可选择文档类型。
- 可选择多个标签。
- 可填写文档描述。
- 记录上传人、上传时间、文件大小、文件类型。
- 文件保存到服务器本地目录。
- 数据库保存元数据。

支持文件类型：

- PDF。
- Word。
- Excel。
- 图片。
- TXT。
- 其他课程允许的常见格式。

预览建议：

- PDF 使用浏览器内嵌预览。
- 图片在页面中展示。
- TXT 读取文本后展示。
- Word、Excel、PPT 可不支持在线预览，提供下载按钮。

## MVP 范围

第一阶段目标是形成可运行、可演示、可扩展的基础版本，完成“登录进入系统、上传文档、管理文档、分类标签、搜索查看、下载文件”的闭环。

MVP 必须包含：

- 登录、退出、Session 保存登录状态。
- 未登录拦截、管理员和普通用户基础区分。
- 文档列表、详情、编辑、删除。
- 上传 PDF、图片、Word、Excel、TXT 等文件。
- 下载已上传文档。
- 分类新增、修改、删除、列表展示。
- 标签新增、修改、删除、文档绑定标签。
- 按标题、分类、标签进行基础搜索。
- 首页、菜单、列表页、表单页基础布局。
- 基础操作记录：登录、上传、删除。

## 非功能要求

性能：

- 支持至少 5 名用户同时在线。
- 文档列表分页，默认每页 10 条或 20 条。
- 查询条件尽量使用数据库索引字段。
- 单次上传文件大小建议限制为 20MB 至 50MB。
- 首页统计使用简单 SQL 聚合，避免复杂实时计算。

安全：

- 密码不能明文存储，建议使用加密或哈希。
- 普通用户不能访问其他用户私有文档。
- 限制文件类型和文件大小。
- 上传文件名必须重命名，避免路径穿越和覆盖。
- 下载文件时必须权限校验。
- 后台管理功能仅管理员可访问。
- 表单提交需要参数校验。
- 数据库操作使用预编译，防止 SQL 注入。

文件存储：

- 上传文件保存到服务器本地磁盘。
- 文件保存时使用唯一文件名。
- 保留原始文件名用于页面展示和下载。
- 文件目录可按日期和用户划分，例如 `uploads/2026/06/user_1/`。
- 删除文档优先逻辑删除。
- 彻底删除时同步处理本地文件。
- 系统需要处理文件不存在、路径错误、下载失败等异常。
